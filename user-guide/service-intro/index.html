服务

更新时间: 2017-8-2

什么是服务？

在 Basement 中，服务被定义为 一组功能相关的 HTTP 接口的集合。

例如 Render 是一个页面渲染托管服务，这个服务就包括了发布页面，设置域名，查询发布历史等一些列的接口，这些接口都是有一定关联的，所以把它们都放在 Render 这个服务中。

Basement 内置了 Render（高性能的页面渲染托管）、DB（轻量 mongodb 数据存储）和 File（静态资源托管）等稳定高效的基础服务，同时也支持服务提供者在 Basement 上发布自己的服务。

TO 服务使用者

1. 为什么要使用 Basement 的服务？

  1. 只需 Basement 服务账号，开箱即用
  2. 高质量文档
  3. 统一 RESTFUL 接口，易于上手
  4. 生产环境和办公环境皆可使用
  5. 重点服务保证高性能、高可用

2. 如何查看和使用所有服务？

通过 Basement > 任意你所在的产品 > 服务 查看服务列表

服务列表中的所有服务均包含使用文档链接，可以通过该链接获取使用帮助。你既可以调用原始的 HTTP 接口，也可以调用服务开发者提供的各种 SDK 接口。

如果觉得文档写的太烂看，请尽情骚扰服务开发者，为你提供人工的智能帮助！

3. 如何查看服务密钥

通过 Basement > 任意你所在的产品 > 设置 > 服务密钥

TO 服务开发者

1. 为什么要在 Basement 上注册自己的服务？

  1. 提供统一的账户体系和授权管理方案，统一的流量管控等贴心服务，让服务开发者可以专注于实现自己的业务逻辑。
  2. 超简单的服务注册流程，一键式发布服务。
  3. 集中的服务展示，让你的服务快速被更多人所知晓。

2. 如何在 Basement 上注册和管理服务？
